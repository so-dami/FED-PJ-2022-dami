[ JS 혹은 jQuery 라우터 구현 시 로드 불일치 문제 ]

JS 기능이 들어간 페이지의 로드구역 설정 시 본 페이지가 index페이지에 바로 불려질 경우 이 문제는 발생하지 않는다.
그러나 라우터로 서브 로딩을 구현할 경우 이 구역은 라우터 모체가 로딩되는 것으로 실행된다.

따라서, 본 모듈이 적용하려고 한 의도와 달리 본 소스에는 적용되지 못한다.
-> 라우터 로딩 불일치 문제


[ 해결 방법 ]

로딩구역을 함수로 만들고 컴포넌트 소스 하단에서 호출한다.
-> 이 방법은 컴포넌트 소스가 로딩된 후 JS소스 로딩구역을 실행하므로 본 소스에 적용됨을 보장하는 코딩법이다.

ex. function jqFn(){
        로드구역을 포함한 소스
    }
    
    function 컴포넌트(){
        return(

            <>
                소스
                {jqFn()} -> 함수 호출
            </>
            
        ); 
    }

    
[ 기타 해결 방법 ]

리액트 컴포넌트에 onClick 등의 이벤트를 직접 생성하여 이벤트 파트는 리액트에 맡긴다.
그리고 메서드를 컴포넌트 안에 생성하여 jQuery, JS 방법 등을 이용하여 코딩을 완성한다.

이 방법도 확실한 방법이지만 일반적인 코딩 기법과 결합 시에 유지보수가 어렵거나 혼란이 야기될 가능성이 크다.
